                \vspace{1.2em}

                It is well known that:


                $$
                    \text{Path-Connected} \;\Rightarrow\; \text{Connected}
                $$


                The \textbf{converse does not hold}. The topologist's sine curve is the classical counterexample.
            \end{block}

            \begin{block}{Introduction \& Setup}
                The curve $S = \{(x,\sin(1/x)) : x > 0\}$ oscillates infinitely as $x \to 0$.
                Adding the origin produces the space $T = S \cup \{(0,0)\}$.

                We formalize these sets as subsets of $\mathbb{R}^2$, in Lean:

                \begin{lstlisting}
def pos_real := Ioi (0 : ℝ) -- (0, ∞)
noncomputable def sine_curve := fun x ↦ (x, sin (x⁻¹))
def S : Set (ℝ × ℝ) := sine_curve '' pos_real
def Z : Set (ℝ × ℝ) := { (0, 0) }
def T : Set (ℝ × ℝ) := S ∪ Z
    \end{lstlisting}

                \begin{figure}

                                axis equal image=false,
                                xmin=-0.03, xmax=0.27,

                        \end{axis}

            \end{block}

            \begin{block}{Part 1: $T$ is Connected}
                $S$ is connected as the continuous image of $(0,\infty)$.
                \texttt{IsConnected.subset\_closure} from \textbf{Mathlib} captures the fact that a subset between a connected set and its closure is connected
                ($S \subseteq T \subseteq \overline{S}$).
                While the inclusion $S \subseteq T$ is trivially closed by the \texttt{tauto\_set} tactic,
                establishing $T \subseteq \overline{S}$ requires to prove $(0,0) \in \overline{S}$.
                We achieve this by showing the sequence
                $f(n) = \bigl(\tfrac{1}{n\pi},\, 0\bigr) \in S$ converges to the origin (via \textbf{Filter} \texttt{Tendsto} ):

                \begin{lstlisting}
let f : ℕ → ℝ × ℝ := fun n => ((n * Real.pi)⁻¹, 0)
lemma T_sub_cls_S : T ⊆ closure S := ... -- via Tendsto f

theorem T_is_conn : IsConnected T :=
  IsConnected.subset_closure (isConnected_Ioi.image sine_curve <|
    continuous_id.continuousOn.prodMk <|
    Real.continuous_sin.comp_continuousOn <|
    ContinuousOn.inv₀ continuous_id.continuousOn
    (fun _ hx => ne_of_gt hx)) (by tauto_set) T_sub_cls_S
    \end{lstlisting}
            \end{block}
        \end{column}

            \begin{block}{Part 2: $T$ is Not Path-Connected}
                If a path connects $S$ to $Z$, its $y$-coordinate
                must oscillate infinitely between $-1$ and $1$ as its $x$-coordinate approaches $0$.
                This infinite oscillation contradicts the continuity of the path.

                We formalize this by contradiction.
                In Mathlib, path-connectedness is defined via \texttt{JoinedIn}:
                \begin{lstlisting}
def IsPathConnected (F : Set X) : Prop :=
  ∃ x ∈ F, ∀ ⦃y⦄, y ∈ F → JoinedIn F x y

def JoinedIn (S : Set X) (x y : X) : Prop :=
  ∃ γ : Path x y, ∀ t, γ t ∈ S
        \end{lstlisting}

                We assume $T$ is path-connected and extract a continuous path
                $p : [0,1] \to T$ between $(0,0)$ and $(1, \sin 1)$:

                \begin{lstlisting}[language=lean]
theorem T_is_not_path_conn : ¬ (IsPathConnected T) := by
  intro p_conn
  let p := JoinedIn.somePath p_conn
        \end{lstlisting}

                We define $t_0$ as the \textit{last time} the path lies on the $y$-axis
                (using \texttt{sSup} rather than the \texttt{sInf} used in K.\ Conrad's paper [1] simplifies later arguments):
                \begin{lstlisting}
  let t₀ : unitInterval := sSup {t | (p t).1 = 0}
        \end{lstlisting}

                By continuity at $t_0$, for $\varepsilon = 1/2$, there exists $\delta > 0$ such that the path stays inside a ball of radius $1/2$ \textbf{\textcolor{UniBlue}{[a]}}:

                \begin{lstlisting}
  obtain ⟨δ, hδ, ht⟩ : ∃ δ > 0, ∀ t,
    dist t t₀ < δ → dist (p t) (p t₀) < 1/2
        \end{lstlisting}

                We find a time $t_1 \in (t_0, t_0 + \delta)$.
                Since $t_1 > t_0$ and $t_0$ is the \emph{last} time on the $y$-axis,
                $p(t_1)$ must lie in $S$ (if it were in $Z$, then $t_1 \leq t_0$, a contradiction).
                This gives $a = x(p(t_1)) > 0$.

                The image $x(p([t_0, t_1]))$ is connected and contains
                $0 = x(p(t_0))$ and $a = x(p(t_1))$.

                Since every connected subset of $\mathbb{R}$ is an interval,
                $[0, a] \subseteq x(p([t_0, t_1]))$ \textbf{\textcolor{UniBlue}{[b]}}:

                \begin{lstlisting}[language=lean]
  have Icc_of_a_b_sub_Icc_t₀_t₁: 
    Set.Icc 0 a ⊆ xcoord_path '' Set.Icc t₀ t₁ :=
      IsConnected.Icc_subset ...
        \end{lstlisting}

            \begin{block}{The Contradiction}
                To cover $[0,a]$, the path must traverse the infinite oscillations. We isolate the peaks where $y=1$ by defining the sequence $x_k = \frac{2}{(4k + 1)\pi}$:

                \begin{lstlisting}
noncomputable def xs_pos_peak :=
  fun (k : ℕ) => 2 / ((4 * k + 1) * Real.pi)
\end{lstlisting}

                \begin{figure}
                    \centering
                    \begin{tikzpicture}
                        \begin{axis}[
                                sine axis,
                                axis equal image=false,
                                xmin=-0.03, xmax=0.25,
                                ymin=0.0, ymax=1.25,
                                height=0.35\linewidth
                            ]
                            \addplot [CurveBlue!50, thin, domain=0.1:0.25, samples=150] {sin(deg(1/x))};
                            \addplot [CurveBlue!50, thin, domain=0.005:0.1, samples=850] {sin(deg(1/x))};

                            \draw[dotred, dashed, line width=3pt] (-0.03, 1) -- (0.25, 1);
                            \node[above left, dotred, font=\small] at (0, 1.0) {$1$};

                            \addplot [CurveBlue!50, mark=*, mark size=5pt] coordinates {(0,0)};

                            \addplot [dotred, mark=*, mark size=5pt, only marks] coordinates {
                                    (0.1273, 1) (0.0707, 1) (0.0489, 1)
                                    (0.0374, 1) (0.0303, 1) (0.0254, 1) (0.0219, 1) (0.0192, 1)
                                };
                        \end{axis}
                    \end{tikzpicture}
                    \caption{The sequence of peaks \texttt{xs\_pos\_peak} isolating $y = 1$.}
                \end{figure}
                As $k \to \infty$, these peaks approach $0$ and enter $[0, a]$. Since the path covers this interval (\textbf{\textcolor{UniBlue}{[b]}}), there must be a time $t' \in [t_0, t_1]$ where the path hits a peak:

                \begin{lstlisting}
obtain ⟨t', ht', hpath_t'⟩ : 
  ∃ t' ∈ Set.Icc t₀ t₁, (p t').2 = 1 := ...
\end{lstlisting}

                Because $t' \in [t_0, t_1]$ (so $t'$ is within $\delta$ of $t_0$), our earlier continuity argument (\textbf{\textcolor{UniBlue}{[a]}}) requires the distance to the origin to be $< 1/2$. However, using \texttt{calc}, we prove the distance must be at least $1$ since the $y$-coordinate is $1$:

                \begin{lstlisting}[language=lean]
have far : 1 ≤ dist (p t') (p t₀) := by
  calc 1 = |(p t').2 - (p t₀).2| := by simp [hpath_t', hpt₀]
       _ ≤ ‖p t' - p t₀‖         := norm_ge_abs_snd
       _ = dist (p t') (p t₀)    := by rw [dist_eq_norm]
\end{lstlisting}

                This contradiction proves $T$ cannot be path-connected.
            \end{block}

            \begin{block}{Conclusion}
                Combining both arguments yields the final theorem:

                \begin{lstlisting}
theorem T_is_conn_not_pathconn :
  IsConnected T ∧ ¬IsPathConnected T := ⟨T_is_conn, T_is_not_path_conn⟩
    \end{lstlisting}

                A stronger version (which considers $T = \overline{S}$) of this formalization has been \textbf{merged into
                    Mathlib} as part of the official counterexamples library:\\
                \texttt{Mathlib/Counterexamples/TopologistsSineCurve}

                \vspace{0.5em}

                This formalization strictly follows K.\ Conrad's paper [1].
                Mathlib provided tools for translating this informal reasoning:
                \texttt{T\_is\_not\_path\_conn} directly uses contradiction,
                Filters (like \texttt{Tendsto}) intuitively handle limits, and the metric space
                \texttt{dist} was conveniently used.
                However, working with the \texttt{unitInterval} subtype required cumbersome explicit casting.
