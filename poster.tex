            \begin{block}{Introduction \& Setup}
                The \textbf{topologist's sine curve} is a classical counterexample in topology illustrating that:


                $$
                    \text{Connected} \;\not\Rightarrow\; \text{Path Connected}.
                $$


                We define the sets as subsets of $\mathbb{R}^2$:

                \begin{lstlisting}
def pos_real := Ioi (0 : ℝ) -- (0, ∞)
noncomputable def sine_curve := fun x ↦ (x, sin (x⁻¹))
def S : Set (ℝ × ℝ) := sine_curve '' pos_real
def Z : Set (ℝ × ℝ) := { (0, 0) }
def T : Set (ℝ × ℝ) := S ∪ Z
\end{lstlisting}

                \begin{figure}
                    \centering
                    \begin{tikzpicture}
                        \begin{axis}[sine axis]
                            \addplot [CurveBlue, thick, domain=0.1:1.15, samples=150] {sin(deg(1/x))};
                            \addplot [CurveBlue, thick, domain=0.005:0.1, samples=850] {sin(deg(1/x))};
                            \addplot [dotred, mark=*, mark size=2.5pt] coordinates {(0,0)};

                            \draw[gray, thick] (-0.05, 1) -- (0.05, 1);
                            \node[left, font=\small, text=gray] at (-0.05, 1) {$1$};
                            \draw[gray, thick] (-0.05, -1) -- (0.05, -1);
                            \node[left, font=\small, text=gray] at (-0.05, -1) {$-1$};
                        \end{axis}
                    \end{tikzpicture}
                    \caption{Graph of $T$}
                \end{figure}

                The curve $S = \{(x,\sin(1/x)) : x > 0\}$ oscillates infinitely as $x \to 0$. Adding the origin produces the space $T = S \cup \{(0,0)\}$.
            \end{block}

            \begin{block}{Part 1: $T$ is Connected}
                $S$ is connected because it is the continuous image of $(0, \infty)$.
                Since every subset between a connected space and its closure is connected
                ($S \subseteq T \subseteq \overline{S}$), $T$ must also be connected.

                To prove $(0,0) \in \overline{S}$, we construct the sequence $f(n) = (\frac{1}{n\pi}, 0)$ and with
                \texttt{Tendsto}, we prove the closure limit:
                \begin{lstlisting}
let f : ℕ → ℝ × ℝ := fun n => ((n * Real.pi)⁻¹, 0)
lemma T_sub_cls_S : T ⊆ closure S := ... -- via Tendsto f
\end{lstlisting}
                We compress the argument using
                \texttt{IsConnected.subset\_closure}, requiring three proofs:
                \begin{itemize}
                    \item \textbf{$S$ is connected:} Chaining continuity lemmas via (\texttt{<|}).
                    \item \textbf{$S \subseteq T$:} Solved trivially by \texttt{tauto\_set}.
                    \item \textbf{$T \subseteq \overline{S}$:} Provided by our sequence lemma.
                \end{itemize}

                \begin{lstlisting}[language=lean]
theorem T_is_conn : IsConnected T :=
  IsConnected.subset_closure (isConnected_Ioi.image sine_curve <|
    continuous_id.continuousOn.prodMk <|
    Real.continuous_sin.comp_continuousOn <|
    ContinuousOn.inv₀ continuous_id.continuousOn
    (fun _ hx => ne_of_gt hx)) (by tauto_set) T_sub_cls_S
\end{lstlisting}
            \end{block}
        \end{column}

        \begin{column}{0.31\textwidth}
            \begin{block}{Part 2: $T$ is Not Path-Connected}
                If a path connects $S$ to $Z$, its $y$-coordinate
                must oscillate infinitely between $-1$ and $1$ as its $x$-coordinate approaches $0$.
                This infinite oscillation contradicts the continuity of the path.
                We formalize this by contradiction:

                \begin{lstlisting}[language=lean]
theorem T_is_not_path_conn : ¬ (IsPathConnected T) := by
  intro p_conn
  -- Extract a continuous path p : [0,1] -> T 
  let p := JoinedIn.somePath p_conn
\end{lstlisting}

                We define $t_0$ as the \textit{last time} the path lies on the $y$-axis:
                \begin{lstlisting}
  let t₀ : unitInterval := sSup {t | (p t).1 = 0}
\end{lstlisting}

                By continuity at $t_0$, for $\varepsilon = 1/2$, there exists a $\delta > 0$ such that the path remains strictly inside a ball of radius $1/2$:

                \begin{lstlisting}
  obtain ⟨δ, hδ, ht⟩ : ∃ δ > 0, ∀ t,
    dist t t₀ < δ → dist (p t) (p t₀) < 1/2
\end{lstlisting}

                Immediately after $t_0$, the path enters $S$.
                We find a time $t_1 > t_0$ in the $\delta$-neighborhood,
                giving a point $a = x(p(t_1)) > 0$, where $x$ is the $x$-coordinate
                function.
                The image $x(p([t_0, t_1]))$ is connected and contains $0 = x(p(t_0))$
                and $a = x(p(t_1))$. Since every connected subset of $\mathbb{R}$ is an interval,
                $[0, a] \subseteq x(p([t_0, t_1]))$.

                \begin{lstlisting}[language=lean]
  have Icc_of_a_b_sub_Icc_t₀_t₁: 
    Set.Icc 0 a ⊆ xcoord_path '' Set.Icc t₀ t₁ :=
      IsConnected.Icc_subset ...
\end{lstlisting}

        % ------------------------------------------------
        \begin{column}{0.31\textwidth}

            \begin{block}{The Contradiction}
                To cover $[0,a]$, the path must traverse the infinite oscillations. We isolate the peaks where $y=1$ by defining the sequence $x_k = \frac{2}{(4k + 1)\pi}$:

                \begin{lstlisting}
noncomputable def xs_pos_peak :=
  fun (k : ℕ) => 2 / ((4 * k + 1) * Real.pi)
\end{lstlisting}

                \begin{figure}
                    \centering
                    \begin{tikzpicture}
                        \begin{axis}[sine axis]
                            \draw[gray, dashed, thick] (-0.55, 1) -- (1.15, 1);
                            \node[above, gray] at (-0.4, 1.0) {1};

                            \addplot [CurveBlue, thick, domain=0.1:1.15, samples=150] {sin(deg(1/x))};
                            \addplot [CurveBlue, thick, domain=0.005:0.1, samples=850] {sin(deg(1/x))};
                            \addplot [dotred, mark=*, mark size=2.5pt] coordinates {(0,0)};

                            \addplot [dotred, mark=*, mark size=1.8pt, only marks] coordinates {
                                    (0.6366, 1) (0.1273, 1) (0.0707, 1) (0.0489, 1)
                                    (0.0374, 1) (0.0303, 1) (0.0254, 1) (0.0219, 1) (0.0192, 1)
                                };
                        \end{axis}
                    \end{tikzpicture}
                    \caption{The sequence of peaks \texttt{xs\_pos\_peak} isolating $y = 1$.}
                \end{figure}

                As $k \to \infty$, these peaks approach $0$ and enter $[0, a]$. Since the path covers this interval, there must be a time $t' \in [t_0, t_1]$ where the path hits a peak:

                \begin{lstlisting}
obtain ⟨t', ht', hpath_t'⟩ : 
  ∃ t' ∈ Set.Icc t₀ t₁, (p t').2 = 1 := ...
\end{lstlisting}

                Because $t' \in [t_0, t_1]$, our earlier continuity argument requires the distance to the origin to be $< 1/2$. However, using \texttt{calc}, we prove the distance must be at least $1$ since the $y$-coordinate is $1$:

                \begin{lstlisting}[language=lean]
have far : 1 ≤ dist (p t') (p t₀) := by
  calc 1 = |(p t').2 - (p t₀).2| := by simp [hpath_t', hpt₀]
       _ ≤ ‖p t' - p t₀‖         := norm_ge_abs_snd
       _ = dist (p t') (p t₀)    := by rw [dist_eq_norm]
\end{lstlisting}

                This contradiction proves $T$ cannot be path-connected.
            \end{block}

            \begin{block}{Conclusion \& Reflections}
                Combining both arguments yields the final theorem:

                \begin{lstlisting}
theorem T_is_conn_not_pathconn :
  IsConnected T ∧ ¬IsPathConnected T := 
  ⟨T_is_conn, T_is_not_path_conn⟩
\end{lstlisting}

                The formalization strictly follows K. Conrad's paper \textit{Connected but not Path-Connected}. Mathlib largely mirrors this informal reasoning:
                \texttt{T\_is\_not\_path\_conn} naturally uses contradiction,
                filters (\texttt{Tendsto}) intuitively handle limits, and the metric space
                \texttt{dist} was conveniently used.

                Working with the \texttt{unitInterval} subtype, required cumbersome explicit casting.

                Ultimately, these compiling lines stand as a verified witness, showcasing how formal reasoning
                makes mathematics rigorous and automatically verifiable.
